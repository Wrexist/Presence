# SETUP тАФ Presence

> How to get from a fresh clone to a building app тАФ including from a Windows-only machine.

---

## TL;DR

- **Develop on Windows.** All Swift source lives under `Presence/` and you edit it in any editor (VS Code, Cursor, etc.).
- **No `.xcodeproj` is committed.** It's generated from `project.yml` by [XcodeGen](https://github.com/yonaskolb/XcodeGen).
- **GitHub Actions builds it for you** on macOS runners (`.github/workflows/pr-checks.yml`). Push to a branch тЖТ see whether it builds.
- **You only need a Mac to run on a real device or submit to TestFlight / the App Store.**

## Working on Windows

### Recommended editor
- **VS Code** or **Cursor** with the **Swift extension** by Swift Server Work Group. Gives you syntax highlighting and basic completion. Type checking is best-effort without a Mac тАФ CI is your real type checker.

### Daily loop
1. Edit Swift files under `Presence/` in your editor.
2. Commit and push to a feature branch.
3. GitHub Actions runs `pr-checks.yml`:
   - Installs XcodeGen + SwiftLint
   - Runs `xcodegen generate`
   - Runs SwiftLint (strict)
   - Builds for iOS Simulator
   - Runs unit tests
4. Read the green/red on your PR. Fix and push again.

### What you can't do on Windows
- Run the app in a simulator.
- Run UI tests against a live simulator.
- Submit to TestFlight / App Store.

For these you need either:
- A real Mac (Mac mini ~$600 is the cheapest path),
- A rented cloud Mac ([MacInCloud](https://www.macincloud.com), [MacStadium](https://www.macstadium.com)), or
- A friend with a Mac who'll let you `ssh` in for an hour when you need to test on device.

## Working on macOS (when you eventually have access)

Prerequisites:
- macOS 15 or newer
- Xcode 26.0 or newer (from the Mac App Store)
- [Homebrew](https://brew.sh)

```bash
# One-time install
brew install xcodegen swiftlint

# Generate the Xcode project
xcodegen generate

# Open it
open Presence.xcodeproj
```

Hit тЦ╢ in Xcode. The app should launch in the iPhone 16 simulator and show the placeholder root view.

### Re-generating the project
Any time you add a new file, change a Swift Package dependency, or edit `project.yml`, run:
```bash
xcodegen generate
```
This is destructive тАФ it overwrites `Presence.xcodeproj`. That's fine because the project file is gitignored.

## Environment variables

Copy `.env.example` to `.env.development` and fill in:
```
SUPABASE_URL=https://[your-project].supabase.co
SUPABASE_ANON_KEY=[anon key]
BACKEND_URL=http://localhost:3000
ANTHROPIC_API_KEY=[backend only тАФ never in client bundle]
REVENUECAT_API_KEY=[ios sdk key]
```

The Anthropic API key is read by the **backend only** тАФ it must never appear in the iOS bundle. See `CLAUDE.md` ┬з "AI / Claude API Integration" for why.
